# 크루즈 차량 데이터 입력 지침

## 목적

크루즈 차량 가격을 `rentcar_price`에 추가할 때

- 잘못된 `way_type` 저장을 막고
- `cruise='공통'` 오입력으로 모든 크루즈에 노출되는 사고를 막고
- 부족한 데이터를 먼저 사용자에게 확인받은 뒤
- 재실행 가능한 SQL 파일을 작성하기 위한 운영 지침이다.

---

## 핵심 규칙

### 1. 저장 테이블

크루즈 차량 가격은 `rentcar_price` 테이블에 저장한다.

- 기준 코드: `rent_code`
- 크루즈별 식별: `cruise`
- 화면 표시 주요값: `way_type`, `route`, `vehicle_type`, `price`
- 예약 연결: `reservation_cruise_car.rentcar_price_code -> rentcar_price.rent_code`

### 2. `cruise='공통'` 사용 금지

크루즈 전용 차량은 `cruise='공통'`으로 넣지 않는다.

이유:

- 공통 데이터는 여러 크루즈 차량 선택 화면에 함께 노출될 수 있다.
- 특정 크루즈 전용 셔틀이 다른 크루즈에도 보이는 문제가 생긴다.

규칙:

- 전용 차량이면 `cruise = 정확한 크루즈명`
- 정말 모든 크루즈에서 공통 사용 차량일 때만 `공통` 허용

### 3. `way_type` 규칙

크루즈 일정 기준으로 `way_type`을 정한다.

- `DAY` / 당일투어 왕복: `당일왕복`
- `1박2일`, `2박3일` 등 숙박 포함 왕복: `다른날왕복`
- 편도 이동만 판매: `편도`

주의:

- `1박2일` 크루즈의 왕복 차량을 `당일왕복`으로 넣으면 안 된다.
- 하선일 복귀가 포함되면 기본적으로 `다른날왕복`이다.

### 4. `rental_type` 규칙

기본값은 아래 원칙을 따른다.

- 차량명 또는 카테고리에 `셔틀` 포함: `공유차량`
- 그 외 단독 차량: `단독대여`

예시:

- `크루즈 셔틀 리무진` -> `공유차량`
- `11인승 리무진` 단독 배차 -> `단독대여`

### 5. `route` 규칙

UI 및 조회 로직에서 경로 문자열을 직접 사용하므로 표준 경로명을 쓴다.

권장 형식:

- `하노이 - 하롱베이`
- `하롱베이 - 하이퐁`
- `하롱베이 - 닌빈`

주의:

- `돌핀 하롱 크루즈 이동`처럼 크루즈명 위주의 비표준 route는
  일부 화면 필터에 걸리지 않을 수 있다.

### 6. `vehicle_type` 규칙

사용자 화면에 직접 보이는 값이므로 통일된 명칭을 쓴다.

권장 예시:

- `승용차`
- `SUV`
- `카니발`
- `9인승 리무진`
- `11인승 리무진`
- `크루즈 셔틀 리무진`

## SQL 작성 전 필수 확인 질문

크루즈 차량 추가 요청이 오면 아래 항목이 하나라도 비어 있으면
SQL부터 쓰지 말고 먼저 사용자에게 물어본다.

### A. 크루즈 식별

1. 정확한 크루즈명이 무엇인가?
2. 이 차량이 해당 크루즈 전용인가, 여러 크루즈 공통인가?

### B. 일정/이용방식

3. 크루즈 일정이 `당일 / 1박2일 / 2박3일` 중 무엇인가?
4. 차량 상품이 `편도 / 왕복` 중 무엇인가?
5. 왕복이면 `당일왕복`인지 `다른날왕복`인지 확정되었는가?

### C. 차량 정보

6. 사용자에게 보일 차량명(`vehicle_type`)은 무엇인가?
7. 최대 탑승 인원(`capacity`)은 몇 명인가?
8. `공유차량`인지 `단독대여`인지 확정되었는가?

### D. 경로 정보

9. 출발지(`route_from`)는 어디인가?
10. 도착지(`route_to`)는 어디인가?
11. 표준 `route` 문자열은 무엇인가?

### E. 가격 정보

12. 편도 가격은 얼마인가?
13. 왕복 가격은 얼마인가?
14. 연도(`year`)는 몇 년 기준인가?

### F. 운영 정책

15. 재실행 시 중복 삽입 방지를 위해 고유 `rent_code` 규칙이 정해졌는가?
16. 기존 동일 크루즈 차량 데이터가 있으면 유지인지, 교체인지, 비활성화인지 확인되었는가?

---

## 사용자에게 되물어야 하는 상황

아래 경우에는 반드시 질문 후 진행한다.

- “왕복”이라고만 말하고 `당일왕복/다른날왕복`이 안 적혀 있는 경우
- 크루즈명이 `아테나/아테네`처럼 혼용되는 경우
- `공통으로 넣어달라`는 요청이지만 실제로는 특정 크루즈 전용으로 보이는 경우
- `크루즈 셔틀 리무진`인데 `공유차량/단독대여`가 명시되지 않은 경우
- 경로가 `크루즈 이동`, `셔틀 이동`처럼 UI 표준 경로가 아닌 경우
- 가격이 편도만 있고 왕복이 없는 경우

## SQL 작성 규칙

### 1. 파일명

아래 형식을 권장한다.

`{번호}-{cruise-slug}-vehicle-{요약}-{YYYYMMDD}.sql`

예:

- `108-athena-premium-cruise-shuttle-limousine-20260621.sql`

### 2. INSERT 원칙

- `public.rentcar_price` 대상
- `WHERE NOT EXISTS`로 중복 방지
- `rent_code`는 의미 있는 고유값 사용
- 가능하면 `BEGIN` / `COMMIT` 포함

### 3. 기본 컬럼 체크리스트

- `rent_code`
- `category`
- `car_category_code`
- `vehicle_type`
- `route`
- `route_from`
- `route_to`
- `way_type`
- `price`
- `capacity`
- `rental_type`
- `year`
- `cruise`
- `memo`
- `description`
- `is_active`

### 4. 검증 쿼리 포함

파일 끝에는 최소 아래 검증이 있어야 한다.

- 추가된 `rent_code` 1건 또는 2건 조회
- `vehicle_type`, `way_type`, `route`, `price`, `cruise`, `rental_type` 확인

## 표준 판단 예시

### 예시 1. 1박2일 크루즈 셔틀 리무진

- 크루즈명: `아테나 프리미엄 크루즈`
- 차량명: `크루즈 셔틀 리무진`
- 편도: `550000`
- 왕복: `1000000`
- 일정: `1박2일`

정답:

- 편도 행 `way_type = '편도'`
- 왕복 행 `way_type = '다른날왕복'`
- `rental_type = '공유차량'`
- `cruise = '아테나 프리미엄 크루즈'`

### 예시 2. 특정 크루즈 전용 단독 11인승

- 차량명: `11인승 리무진`
- 운영 방식: 단독 배차

정답:

- `rental_type = '단독대여'`

## 최종 완료 조건

크루즈 차량 추가 작업 완료는 아래를 모두 만족해야 한다.

- 부족한 입력값을 사용자에게 확인했다
- `way_type`이 일정과 맞다
- `cruise='공통'` 오입력이 아니다
- `rental_type`이 셔틀/단독 정책과 맞다
- SQL이 재실행 가능하다
- 검증 쿼리가 파일에 포함되어 있다
