## 11) 추가 정밀 점검 결과 (원인 확정)
- 결론
  - 호텔 목록 미표시의 직접 원인은 고객앱 쿼리가 날짜 조건에 강하게 의존하는 구조입니다.
  - 요코온센 작업(118)은 요코 중심 보정이라 전체 호텔 커버리지 공백을 동시에 해결하지 못했고, 그 시점에 문제가 표면화되었습니다.

- 고객앱 로딩 구조
  - 1단계. hotel_price에서 체크인 날짜 조건(start_date <= checkin <= end_date)으로 hotel_code 조회.
  - 2단계. 조회된 hotel_code만 hotel_info(active=true)와 매칭해서 호텔 목록 표시.
  - 결과. 특정 날짜에 hotel_price가 비면 hotel_info에 호텔이 있어도 목록은 0개가 됩니다.

- SQL 118 영향
  - 요코 코드 정규화와 요코 2026/2027 보정은 수행함.
  - 하지만 적용 범위가 요코 조건(hotel_code/hotel_name 요코 계열) 중심이라 전체 호텔 문제를 막지 못함.

- SQL 120/121 상태
  - 현재 워크스페이스에는 sql/120-all-hotels-extend-2026-2027-20260705.sql, sql/121-fix-hotel-price-end-date-update-20260705.sql 파일이 존재하지 않음.
  - 과거에 실행되었을 가능성은 있으나, 로컬 파일 기준으로는 현재 재검증 불가 상태.
